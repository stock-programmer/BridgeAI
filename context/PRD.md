## 1. 项目背景与目标

### 1.1 背景
在企业协作中，产品经理（PM）与开发工程师（Dev）常因关注点不同（商业价值 vs 技术实现）导致沟通同频困难。这种认知偏差往往造成需求返工、评估不准或价值传递失败。

### 1.2 目标
开发一款基于大模型的Web端“职能翻译助手”，通过AI将“业务语言”与“技术语言”进行双向深度转译，实现：
1.  **降低沟通成本**：减少反复确认需求细节的会议时间。
2.  **消除认知壁垒**：帮助技术理解业务价值，帮助产品理解技术复杂度。

## 2. 用户角色

| 角色 | 核心痛点 | 期望收益 |
| :--- | :--- | :--- |
| **产品经理 (PM)** | 不懂技术细节，难以评估工作量，需求描述被开发认为“太虚”。 | 获得包含算法建议、数据链路、性能要求的技术化需求文档。 |
| **开发工程师 (Dev)** | 不懂商业逻辑，难以向上汇报技术优化的价值，觉得产品“乱提需求”。 | 获得包含用户体验影响、ROI分析、业务增长空间的价值描述文档。 |

## 3. 页面结构

```
┌─────────────────────────────────────────┐
│         职能沟通翻译助手                  │
│    让产品和开发真正听懂彼此                │
├─────────────────────────────────────────┤
│                                         │
│  [我是产品经理]  [我是开发工程师]  <--- 角色选择  │
│                                         │
│  ┌───────────────────────────────────┐ │
│  │                                   │ │
│  │  请输入你想表达的内容...            │ │
│  │  (支持换行)                         │ │
│  │                                   │ │
│  └───────────────────────────────────┘ │
│                                         │
│              [开始翻译] <--- 提交按钮      │
│                                         │
├─────────────────────────────────────────┤
│  翻译结果：                               │
│                                         │
│  [流式输出区域]                           │
│  - 支持markdown渲染                      │
│  - 打字机效果                            │
│                                         │
│              [复制结果]  [清空]           │
│                                         │
└─────────────────────────────────────────┘
```
交互流程：
1. **用户选择角色** → 界面有文本提示根据角色进行变化
   - 产品：如"描述你的需求或想法"
   - 开发：如"描述你的技术方案或改动"

2. **输入内容** → 实时字数统计（建议50-500字）

3. **点击翻译** →
   - 按钮状态变为"翻译中..."
   - 禁用输入框
   - 开始流式输出

4. **流式展示** →
   - 打字机效果逐字显示

5. **翻译完成** →
   - 显示"复制结果"按钮
   - 恢复输入状态
